# Project Governance

This document describes the default governance model for this project.

## Roles

### Users

Users rely on the project and may report bugs, request features, or participate in discussions.

### Contributors

Contributors submit issues, documentation improvements, tests, code changes, or reviews.

### Maintainers

Maintainers are responsible for project direction, review, releases, security handling, and moderation.

## Decision Making

Maintainers try to make decisions transparently and based on project goals, maintainability, security, user impact, and contributor feedback.

For routine changes, approval through the normal pull request process is sufficient.

For major changes, maintainers may request a design discussion or proposal before implementation.

## Pull Request Merging

Pull requests may be merged when:

- Required reviews are approved.
- Required status checks pass.
- Required conversations are resolved.
- The change is aligned with the project scope.
- Documentation and tests are updated when needed.

Maintainers may close pull requests that are out of scope, inactive, unsafe, or not maintainable.

## Releases

Releases are created by maintainers. Release notes should summarize notable changes, bug fixes, security updates, and breaking changes.

## Security

Security reports are handled according to `SECURITY.md`.

## Changes to Governance

This governance model may evolve as the project grows. Changes should be proposed through a pull request.
