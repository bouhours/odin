# Support

Thank you for using this project.

## Getting Help

Before opening a new issue, please:

1. Read the README and documentation.
2. Search existing issues and discussions.
3. Check whether your problem is already fixed in the latest version.

## Where to Ask

- Bug reports: use the bug report issue template.
- Feature requests: use the feature request issue template.
- General questions: use GitHub Discussions if enabled, or open a question issue.
- Security vulnerabilities: follow the instructions in `SECURITY.md`.

## What to Include

When asking for help, include:

- Project version or commit SHA.
- Operating system and environment.
- Configuration details relevant to the issue.
- Steps to reproduce.
- Logs or error messages.
- What you expected to happen.
- What actually happened.

## Scope of Support

Maintainers and contributors may provide help on a best-effort basis. There is no guaranteed response time unless a separate support agreement exists.
